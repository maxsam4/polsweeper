Data Flow: Create Accounts

User enters master address + count (1-5) in UI
         │
         ▼
Frontend POST /api/create { master: "0x...", count: 3 }
         │
         ▼
Backend validates:
  ✓ count ≤ 5
  ✓ existing + count ≤ 5 (query SQLite)
  ✓ master is valid address
         │
         ▼
Backend calls factory.createAccounts(master, 3)
  → signs tx with CREATE_GAS_ACCOUNT private key
  → waits for confirmation
         │
         ▼
Backend stores in SQLite:
  (master, clone_addr_0, 0, now, tx_hash)
  (master, clone_addr_1, 1, now, tx_hash)
  (master, clone_addr_2, 2, now, tx_hash)
         │
         ▼
Returns addresses to frontend

Data Flow: Manual Sweep

User clicks Sweep on a virtual account
  optionally provides ERC20 token address
         │
         ▼
Frontend POST /api/sweep { account: "0x...", token?: "0xUSDC..." }
         │
         ▼
Backend determines sweep type:
  If token provided → call sweepERC20(token) on clone
  If no token       → call sweepPOL() on clone
         │
         ▼
Backend signs tx with SWEEP_GAS_ACCOUNT
  → waits for confirmation
         │
         ▼
Returns tx hash + swept amount to frontend

Gas Account Separation

CREATE_GAS_ACCOUNT

Funds account creation txns. Budget is predictable (5 accounts max per master × fixed gas per clone).

Env var: CREATE_GAS_PRIVATE_KEY

SWEEP_GAS_ACCOUNT

Funds sweep txns. Usage scales with incoming deposits. Separate so a sweep gas drain doesn't block account creation.

Env var: SWEEP_GAS_PRIVATE_KEY
